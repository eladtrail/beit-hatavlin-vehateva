# Plan 01-04: @layer Architecture + Eliminate !important

## Objective

Restructure both CSS files to use CSS Cascade Layers (`@layer`), which makes the cascade explicit and eliminates the need for `!important`. The three `!important` declarations on `.nav-cta` (lines 293, 296, 299 in style.css) exist because `.nav-cta` fights specificity with `.nav-links a`. Moving `.nav-cta` into the `overrides` layer — which always wins over `components` where `.nav-links a` lives — removes the specificity conflict without any `!important`. This is the highest-risk change in the phase (cascade restructure) and runs last in Wave 2 after all other changes are verified stable.

Eliminating `!important` is a direct consequence of FOUND-01 (@layer) — no standalone requirement ID exists for it. FOUND-06 (unused font weights) and FOUND-07 (CSV timeout) are fully implemented in PLAN-01-03.
